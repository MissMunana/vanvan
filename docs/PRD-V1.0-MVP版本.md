# 小星星成长宝 - V1.0 MVP版本 PRD

## 📋 文档信息

- **产品名称**：小星星成长宝（原名：小星星习惯宝）
- **产品定位**：基于循证育儿理念的一站式家庭育儿助手
- **文档版本**：v2.0（综合版）
- **创建日期**：2026-02-09
- **产品类型**：微信小程序 / H5应用
- **目标用户**：3-12岁儿童及其家长

---

## 🎯 产品概述

### 产品定位升级

**原定位**：儿童习惯养成积分小程序
**新定位**：基于循证育儿理念的一站式家庭育儿助手

从单一的习惯养成工具，升级为覆盖"习惯养成 + 健康管理 + 情绪发展 + 循证育儿知识"的综合育儿助手。

### 核心价值

- **对儿童**：在游戏化体验中养成好习惯，建立自信和责任感
- **对家长**：一站式管理孩子的习惯养成和健康数据，基于科学证据做育儿决策
- **核心理念**：用科学替代焦虑，用数据替代猜测，用鼓励替代说教

### 产品口号

> "科学育儿，看得见的成长"

**成功标准**：

## 核心功能清单

### 1. 习惯养成核心模块（儿童端）

#### 1.1 首页 - 我的积分星球

**功能描述**
- 展示可爱的星球/花园作为主视觉
- 顶部显示当前总积分（大字号、醒目）
- 可视化的积分池（宝箱、储钱罐、星星罐）
- 底部导航：首页、任务、商城、我的

**年龄分层适配机制（P0优先级）**

| 年龄段 | UI复杂度 | 交互方式 | 视觉风格 |
|-------|---------|---------|---------|
| 3-5岁 | 极简：大按钮、少文字、大图标 | 点击为主 | 超可爱卡通风 |
| 6-8岁 | 中等：适度信息展示 | 点击为主，可选拖拽 | 可爱活泼风 |
| 9-12岁 | 丰富：更多数据展示 | 点击+拖拽 | 成长酷炫风 |

**交互设计**
- 积分增加动画：星星/金币从顶部飘落到积分池
- 音效：叮咚声、欢快音效（可关闭）
- 轻微震动反馈（可关闭）

**视觉元素**
- 主题色：温暖的黄色#FFB800
- 吉祥物：可爱的小动物引导角色
- 插画风格：Q版卡通、圆润线条

#### 1.2 任务清单

**功能描述**
- 展示所有可获得积分的任务列表
- 按类型分类：生活习惯🦷、学习习惯📚、礼貌行为😊、家务帮助🧹
- 每个任务显示：图标、名称、积分数、完成按钮

**交互设计（V1.0简化版）**
- **点击加分**（主要方式）：
  1. 点击任务卡片右侧的"✓"按钮
  2. 弹出确认动画
  3. 执行加分动画（星星飞入积分池）
  4. 积分数字增长动画
  5. 播放音效
  6. 任务标记为"已完成"

- **撤销功能**：3秒内可撤销（toast提示）

**连续完成奖励（简化版）**
- 连续3天完成同一任务：额外+5积分
- 连续7天完成同一任务：额外+20积分+特殊动画

**任务卡片设计**
- 大图标（emoji或自定义）
- 任务名称（14-18号字）
- 积分数（大字号、醒目）
- 今日完成状态

#### 1.3 积分商城

**功能描述**
- 展示可兑换的奖励列表
- 奖励分类：
  - 亲子时光（10-100积分）：陪玩桌游、讲故事、一起做手工
  - 小特权（20-50积分）：晚睡15分钟、选择晚餐、多看动画片
  - 实物奖励（100-500积分）：玩具、书籍、文具

**兑换流程**
1. 儿童点击想要兑换的商品
2. 弹出确认弹窗：
   - 商品图片和名称
   - 需要消耗的积分
   - 当前积分是否足够
   - "我想要"按钮
3. 积分足够：提交兑换申请，等待家长审核
4. 积分不足：鼓励性提示"还差X分就够啦！加油！"
5. 申请提交后：显示"等待家长确认"状态

**情绪安全设计（P1优先级）**
- 积分不足时：用鼓励性语言，不使用"失败""不够"等负面词汇
- 兑换被拒时：温和提示"爸爸妈妈觉得可以再等等哦"，并推荐替代奖励
- 失败音效：温和提示音，避免负面联想
- 吉祥物安慰动画：小熊拥抱、鼓励

**商品卡片设计**
- 商品图片（emoji或自定义图片）
- 商品名称
- 所需积分（大字号）
- 进度条：当前积分/所需积分
- 快要够积分的商品有闪光提示

#### 1.4 我的成就（简化版）

**功能描述**
- 积分历史记录（最近30条）
- 本周完成任务统计
- 累计获得积分总数

### 2. 家长管理功能

#### 2.1 家长控制台

**进入方式**
- 首次使用时设置4-6位数字密码
- 从设置页进入家长模式
- 密码错误3次锁定5分钟

**仪表盘内容**
- 本周完成任务数
- 本周积分变化
- 待审核的兑换申请（红点提醒）
- 快捷操作入口

#### 2.2 任务管理

**功能描述**
- 创建、编辑、删除习惯任务
- 快速导入模板（根据年龄推荐）

**创建任务表单**
```
├── 任务名称（必填）
├── 任务分类（下拉选择）
├── 积分奖励（数字输入）
├── 任务图标（emoji选择器）
├── 任务说明（选填）
├── 有效时段（选填）：如"刷牙"只能在7-9点和19-21点完成
└── 是否启用（开关）
```

**任务模板库（P1优先级）**

**学龄前（3-6岁）预置模板**
- 🦷 自己刷牙（10分）
- 👕 自己穿衣服（10分）
- 🧸 收拾玩具（15分）
- 😊 有礼貌说"谢谢"（5分）
- 🗑️ 帮忙扔垃圾（10分）

**小学生（7-12岁）预置模板**
- 📝 按时完成作业（20分）
- 📖 主动阅读30分钟（15分）
- 🎒 整理书包（10分）
- 🧹 帮忙做家务（15分）
- 💬 分享学习心得（10分）

**一键导入功能**
- 家长选择孩子年龄段
- 系统推荐适合的任务模板
- 家长可修改积分值
- 一键批量导入

#### 2.3 商城管理

**功能描述**
- 添加、编辑、删除奖励商品
- 快速导入推荐奖励

**创建商品表单**
```
├── 商品名称（必填）
├── 所需积分（必填）
├── 商品分类（下拉选择）
├── 商品图片（emoji或上传）
├── 兑换限制（每日/每周/每月/不限）
├── 库存设置（无限/限量）
└── 是否上架（开关）
```

**推荐奖励模板**

**亲子时光类（推荐，占比60%）**
- 10分：爸爸妈妈讲睡前故事
- 20分：一起玩桌游30分钟
- 50分：周末去公园玩
- 100分：一起做烘焙

**小特权类**
- 15分：晚睡15分钟
- 20分：选择今天吃什么
- 30分：多看30分钟动画片

**实物类（谨慎使用，占比<20%）**
- 50分：小文具
- 100分：绘本书
- 200分：中等玩具

**积分经济平衡引导（P1优先级）**
- 首次设置时提供建议："日常任务建议5-20分，奖励建议50-500分"
- 系统自动检测积分通胀/通缩，提醒家长调整
- 如孩子积分累积过快（一周获得>500分），提示"积分获得较快，建议适当提高奖励价格或降低任务积分"

#### 2.4 兑换审核

**功能描述**
- 查看孩子的兑换申请列表
- 审核通过或拒绝

**审核流程**
1. 接收到兑换申请通知（App内红点+推送）
2. 查看申请详情
3. 选择操作：
   - **同意**：扣除孩子积分，标记为已兑换
   - **拒绝**：积分不扣除，可填写原因（温和表达）
   - **稍后处理**：暂不操作
4. 操作后推送消息到儿童端

**快捷审核（P1优先级）**
- 通知中直接滑动审核，无需打开App
- 低于50分的兑换自动通过（家长可设置阈值）

#### 2.5 手动调整积分（弱化设计）

**功能描述**
- 家长可手动增加或减少积分
- 用于特殊情况（奖励特殊表现）

**操作流程**
1. 点击"手动调整"按钮
2. 选择操作：增加（推荐）/减少（弱化）
3. 输入积分数值
4. 填写原因说明（必填）
5. 确认提交

**扣分功能的安全设计（P0优先级）**
- 默认隐藏"减少积分"选项
- 点击时弹出教育提示：
  > 💡 心理学研究表明，扣分可能导致孩子对整个系统产生抵触。
  > 建议尝试：与孩子对话了解原因 → 共同制定改进计划 → 用鼓励替代惩罚。
  > 确定要扣分吗？
- 设置扣分上限：单次不超过总积分的10%
- 增加"冷静期"：点击扣分后需等待10分钟才能确认

### 3. 内在动机培养机制（P0优先级）

**理论基础**：自我决定理论（SDT），防止过度依赖外在奖励导致内在动机削弱。

#### 3.1 情绪反思模块

**功能描述**
- 任务完成后，引导孩子反思情绪（3-5秒互动）

**交互流程**
```
任务完成 → 积分动画 → 情绪反思（每3次触发1次）

弹窗内容：
"做完这件事你的感觉怎么样？"
├── 😊 开心
├── 💪 自豪
├── 😌 轻松
└── 🤔 没什么感觉

选择后：
- 吉祥物正向回应："你感到开心，真棒！"
- 记录情绪数据（家长端可查看情绪趋势）
```

**目标**：逐步从"为了积分而做"过渡到"因为做了感到开心而做"

#### 3.2 成就叙事（文案优化）

**原则**：强调过程和努力，弱化积分数字

**文案对比**

| 场景 | ❌ 避免 | ✅ 推荐 |
|-----|--------|--------|
| 完成任务 | "你获得了10积分！" | "你坚持做到了！👏" + 积分提示（小字） |
| 连续完成 | "你已经连续7天了！" | "你已经连续自己刷牙7天了！你的牙齿一定在偷偷感谢你呢！🦷✨" |
| 里程碑 | "你获得了勋章！" | "你真棒！坚持30天，你已经是习惯小达人了！🏆" |

### 4. 通用功能

#### 4.1 注册与登录（P0优先级）

**设计原则**
- 注册流程简洁，降低启动门槛
- 支持多孩家庭在注册阶段同步录入多个孩子信息
- 登录方式优先考虑便捷性和安全性

**注册流程**

```
Step 1：手机号注册
├── 输入手机号
├── 获取短信验证码（60秒倒计时）
├── 输入验证码完成注册
└── 设置家长管理密码（4-6位数字PIN）

Step 2：创建第一个孩子档案
├── 名字：[输入]
├── 性别：[男孩/女孩]
├── 出生日期：[日期选择器]（自动计算年龄和年龄段）
├── 头像：[选择预置头像/上传照片]
└── [继续]

Step 3：是否添加更多孩子（多孩家庭支持）
├── 提示："家里还有其他宝贝吗？"
├── [添加另一个孩子] → 重复Step 2
├── [以后再说] → 跳过，进入下一步
└── 最多支持添加5个孩子

Step 4：为每个孩子选择习惯模板（快速开始）
├── 如有多个孩子，按顺序分别设置
├── 根据每个孩子的年龄推荐不同模板
├── [预置模板列表，默认选中3-5个]
│   □ 自己刷牙 (10分)
│   □ 收拾玩具 (15分)
│   □ 有礼貌问好 (5分)
│   ...
└── [下一步]

Step 5：为每个孩子设置奖励商城
├── 如有多个孩子，可共用同一套奖励或分别设置
├── [推荐奖励列表，默认选中2-3个]
│   □ 讲睡前故事 (10分)
│   □ 一起玩桌游 (20分)
│   □ 周末去公园 (50分)
│   ...
└── [完成，开始使用]

Step 6：30秒动画引导
├── 展示核心流程：完成任务→获得积分→兑换奖励
├── 如有多个孩子，展示切换孩子的操作方法
└── 吉祥物带领互动
```

**登录流程**

```
登录方式（按优先级）：
├── 1. 短信验证码登录（推荐）
│   ├── 输入手机号
│   ├── 获取验证码
│   └── 验证通过后进入首页
│
├── 2. 密码登录（备选）
│   ├── 输入手机号
│   ├── 输入密码
│   └── 验证通过后进入首页
│
└── 3. 微信一键登录（如为小程序版本）
    └── 授权后自动登录

登录后行为：
├── 单孩家庭：直接进入该孩子的首页
├── 多孩家庭：进入上次查看的孩子首页，顶部显示切换入口
└── 首次登录新设备：需验证家长身份（短信验证码）
```

**安全机制**
- 验证码5分钟内有效，连续错误5次锁定30分钟
- 同一手机号每日最多发送10条验证码
- 登录态有效期30天，到期需重新登录
- 敏感操作（修改手机号、注销账号）需二次验证

#### 4.2 首次使用引导

**冷启动体验（P1优先级）**

注册完成后自动进入首次使用引导（注册流程中已包含孩子档案创建、模板选择等步骤，此处为补充性引导）。

#### 4.3 家长教育体系

**新手引导流程（P1优先级）**

首次使用时展示3-5页核心理念：

**第1页：产品理念**
> 小星星成长宝相信：
> 用鼓励替代说教，让好习惯养成充满乐趣！🌟

**第2页：正向激励原则**
> 我们强调"奖励做到的"，而非"惩罚没做到的"。
> 研究表明，正向激励更有助于建立持久的习惯。

**第3页：家长的角色**
> 您的角色是"引导者"而非"监督者"：

> ✅ 关注过程而非结果

**第4页：积分经济建议**
> 💡 小贴士：
> - 建议60%以上的奖励设置为"亲子时光"
> - 物质奖励要适度，避免孩子过度依赖

**第5页：开始使用**

#### 4.4 孩子资料管理（二次编辑）（P1优先级）

**设计背景**
家长在首次注册时可能录入有误，或孩子随时间成长需要更新信息。需要提供便捷的资料编辑入口。

**入口位置**
- 家长控制台 → 孩子档案管理
- 「我的」页面 → 点击孩子头像/名字
- 多孩家庭的孩子切换面板 → 每个孩子旁边的编辑图标

**可编辑字段**

| 字段 | 是否可编辑 | 编辑限制 | 联动影响 |
|-----|-----------|---------|---------|
| 名字 | 可编辑 | 1-10个字符 | 全局显示更新 |
| 性别 | 可编辑 | 仅在家长控制台操作 | 主题推荐更新 |
| 出生日期 | 可编辑 | 需二次确认 | ageGroup重新计算，UI风格/交互方式/模板推荐联动变更 |
| 头像 | 可编辑 | 预置头像或上传照片 | 全局显示更新 |

**年龄变更联动逻辑（重要）**

当修改出生日期导致年龄段（ageGroup）发生变化时：
```
触发联动：
├── UI风格自动切换（如从"超可爱卡通风"→"可爱活泼风"）
├── 交互方式调整（如从"纯点击"→"点击+可选拖拽"）
├── 任务模板推荐更新（弹窗提示："孩子已进入新的年龄段，是否查看推荐的新任务？"）
├── 已有任务和积分数据保留不变
└── 弹出温馨提示："已更新为[6-8岁]模式，页面风格已自动调整"
```

**编辑流程**
```
1. 进入编辑页面（显示当前信息）
2. 修改目标字段
3. 点击"保存"
4. 如涉及出生日期变更：弹出二次确认弹窗
   └── "修改出生日期会影响年龄段设置和界面风格，确定修改吗？"
5. 保存成功：toast提示"已更新"
6. 如年龄段变化：自动应用新的UI风格
```

**添加新孩子（V1.0即支持）**
- 入口：家长控制台 → "添加孩子"按钮
- 流程与注册时创建孩子相同（填写基本信息→选择任务模板→设置商城奖励）
- 最多支持5个孩子

**删除孩子档案**
- 入口：家长控制台 → 孩子档案 → 删除
- 需输入家长密码确认
- 弹出警告："删除后该孩子的所有数据（任务、积分、兑换记录）将无法恢复，确定删除吗？"
- 删除后如无其他孩子，引导创建新孩子档案

**注销账号（合规必需）**

```
入口：家长端设置 → 注销账号（位于设置页最底部，灰色文字）
流程：
├── 1. 点击"注销账号"
├── 2. 展示注销须知：
│   ├── "注销后，以下数据将被永久删除且无法恢复："
│   ├── "- 所有孩子的任务、积分、兑换记录"
│   ├── "- 所有健康数据和成长档案"
│   ├── "- 家长账号信息"
│   └── "- 如有VIP订阅，请先取消自动续费"
├── 3. 输入手机号验证码（身份确认）
├── 4. 输入家长密码（二次确认）
├── 5. 勾选"我已了解注销后果"
├── 6. 点击"确认注销"
├── 7. 进入15天冷静期：
│   ├── 期间账号冻结但不删除数据
│   ├── 冷静期内可通过登录撤销注销
│   └── 15天后正式删除所有数据
└── 8. 注销完成后跳转到登录页面
```

### 6. 技术方案

#### 6.1 技术栈（推荐）

**前端**：微信小程序原生开发 / uni-app
**后端**：Node.js + Express / 微信云开发
**数据库**：微信云数据库 / MongoDB
**存储**：微信云存储（图片、音效）

**优势**
- 用户基数大，易推广
- 开发工具完善
- 云开发降低后端成本
- 审核机制保证安全

#### 6.2 核心技术点

**动画实现**
- 使用CSS3 animation + transition
- 低端设备自动降低动画复杂度
- 提供"流畅模式"选项（关闭动画）

**音效管理**
- 音效库：5-7个核心音效
- 格式：MP3（微信小程序支持）
- 时长：0.5-2秒
- 音量：中等，不刺耳
- 可在设置中关闭

**离线支持**
- V1.0支持离线打卡
- 联网后自动同步
- 避免因网络问题导致无法记录

#### 6.3 安全与合规

**儿童隐私合规（P0优先级）**
- 根据《儿童个人信息网络保护规定》
- 收集信息前获得监护人明确同意
- 数据最小化原则：只收集必要数据（姓名、年龄、性别）
- 不收集位置、通讯录等敏感信息

**数据安全**
- 密码加密存储（bcrypt）
- 接口鉴权（JWT Token）
- HTTPS加密传输
- 定期数据备份

### 7. V1.0成功标准

#### 7.1 产品指标

- 注册用户 > 1000人
- 日活率 > 30%
- 7日留存率 > 40%
- 用户满意度 > 4.0分（5分制）

- 80%的用户至少创建3个习惯任务
- 70%的用户至少完成1次积分兑换
- 单个孩子平均完成任务数 > 2个/天
- 家长审核通过率 > 85%

#### 7.3 习惯养成指标

- 30%的用户养成至少1个稳定习惯（连续7天）
- 60%的用户连续使用超过7天
