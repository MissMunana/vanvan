## 核心功能清单

### 1. 生长曲线管理

#### 1.1 功能概述

基于WHO儿童生长标准和中国儿童生长标准，提供可视化的生长曲线追踪工具。

#### 1.2 数据录入

**测量数据**
- 身高（cm）
- 体重（kg）
- 头围（cm）- 0-3岁
- BMI（自动计算）
- 测量日期

**录入方式**
- 手动输入数字
- 滑动选择器（更直观）
- 可拍照保存体检报告

#### 1.3 生长曲线图表

**参考标准**
- WHO儿童生长标准（0-5岁）
- WHO学龄儿童生长参考（5-19岁）
- 中国儿童生长标准（可选切换）

**展示内容**
- 身高-年龄曲线
- 体重-年龄曲线
- BMI-年龄曲线
- 头围-年龄曲线（0-3岁）

**百分位线展示**
- P3 / P10 / P25 / P50 / P75 / P90 / P97
- 孩子的数据点标注在曲线上
- 手指滑动查看历史数据点

#### 1.4 智能预警

**预警规则**
- 身高/体重低于P3或高于P97：温和提示关注
- 生长速度突然变化（3个月内跨越2条百分位线）：提示评估
- 提示文案："建议带孩子到儿保科进行评估"（不做诊断，只做提示）

**生长速度分析**
- 按月/季度计算生长速度
- 与标准生长速度对比
- 趋势分析：持续偏离提醒

#### 1.5 数据结构

```javascript
// 生长记录表 (growth_records)
{
  recordId: String,
  childId: String,
  date: Date,
  ageInMonths: Number,      // 自动计算
  height: Number,           // cm
  weight: Number,           // kg
  headCircumference: Number, // cm（可选）
  bmi: Number,              // 自动计算
  heightPercentile: Number,  // 自动计算
  weightPercentile: Number,
  bmiPercentile: Number,
  note: String,
  createdAt: Date
}
```

### 2. 发育里程碑追踪

#### 2.1 功能概述

基于WHO和AAP发育里程碑标准，帮助家长了解孩子各项能力的发展情况。

#### 2.2 里程碑类别

**大运动发育**
- 抬头（2-4月）
- 翻身（4-6月）
- 独坐（6-8月）
- 爬行（7-10月）
- 独走（10-15月）
- 跑跳（18-24月）

**精细运动发育**
- 抓握（3-5月）
- 换手拿物（6-8月）
- 拇食指捏取（9-12月）
- 涂鸦（12-18月）
- 搭积木（18-24月）
- 画圆形（30-36月）

**语言发育**
- 发出元音（2-4月）
- 转头寻声（4-6月）
- 叫"爸爸妈妈"（8-12月）
- 说10个词（12-18月）
- 两字短句（18-24月）
- 完整句子（24-36月）

**社交情感发育**
- 社交性微笑（2-3月）
- 分离焦虑（6-12月）
- 模仿行为（9-12月）
- 平行游戏（18-24月）
- 合作游戏（36-48月）

#### 2.3 交互设计

**里程碑卡片**
- 按月龄展示当前应关注的里程碑
- 家长标记"已达成"或"进行中"
- 达成时可拍照/录视频记录（成长档案）
- 如果某项能力显著滞后，温和提示关注并建议咨询儿保医生

**视觉设计**
- 里程碑展示为时间线形式
- 已达成：彩色+勾选
- 未达成：灰色
- 进行中：黄色+进度环

### 3. 健康用药模块

#### 3.1 重要声明

**产品内醒目展示**：
> ⚠️ 本功能提供的所有信息和计算结果仅供参考，不构成医疗建议。
> 儿童用药请务必遵医嘱，具体用药方案以医生处方为准。
> 如孩子出现紧急症状，请立即就医。

#### 3.2 发烧记录与管理

**功能描述**
- 记录孩子的体温变化
- 自动生成体温曲线图
- 用药后体温变化追踪
- 就医提醒

**数据录入**
```javascript
// 体温记录表 (temperature_records)
{
  recordId: String,
  childId: String,
  temperature: Number,      // ℃
  measureMethod: String,    // 耳温/额温/腋温/肛温/口温
  measureTime: Date,
  symptoms: [String],       // 咳嗽/流涕/呕吐/腹泻/皮疹/精神差等
  medication: {             // 是否用药
    used: Boolean,
    drugName: String,
    dosage: String,
    time: Date
  },
  note: String,
  createdAt: Date
}
```

**体温分级与提示（基于AAP指南）**

| 体温范围（腋温） | 分级 | 系统提示 |
|----------------|------|---------|
| 36.0-37.2℃ | 正常 | 体温正常 |
| 37.3-38.4℃ | 低中热 | 关注孩子精神状态，保持正常穿着，少量多次饮水 |
| 38.5-39.9℃ | 中高热 | 如孩子明显不适，可考虑使用退烧药改善舒适度（遵医嘱） |
| ≥40.0℃ | 高热 | 建议使用退烧药并密切观察，如伴有严重症状请立即就医 |

**关于物理降温的循证说明**

| 做法 | 是否推荐 | 依据 |
|------|---------|------|
| 减少过度包裹 | ✅ 合理 | 避免产热增加 |
| 温水擦浴 | ❌ 不推荐 | NICE指南不推荐，可能引起寒颤不适 |
| 酒精擦浴 | ❌ 禁止 | 经皮肤吸收有中毒风险 |
| 退热贴 | ❌ 不推荐 | 无循证依据支持降温效果 |
| 捂汗 | ❌ 禁止 | 可能导致高热惊厥和捂热综合征风险 |
| 多饮水 | ✅ 推荐 | 少量多次补液有助于维持水合状态 |

**就医提醒规则**

以下情况系统强烈建议就医：
- 3月龄以下婴儿体温≥38℃
- 3-6月龄婴儿体温≥39℃
- 发烧超过72小时（3天）
- 体温≥40℃
- 退烧药使用后2小时体温无明显下降
- 伴随症状：精神萎靡、持续呕吐、严重腹泻、皮疹、抽搐、呼吸困难

**体温曲线图**
- 横轴：时间（可查看24h/3天/7天）
- 纵轴：体温（36-42℃）
- 标注用药时间点（药物图标标记）
- 标注38.5℃参考线（虚线）
- 颜色区间：正常（绿）→低热（黄）→中热（橙）→高热（红）

#### 3.3 用药记录

**功能描述**
- 记录每次用药信息
- 用药间隔提醒
- 用药频次统计

**数据结构**
```javascript
// 用药记录表 (medication_records)
{
  recordId: String,
  childId: String,
  drugName: String,
  genericName: String,       // 通用名
  dosageForm: String,        // 混悬液/颗粒/片剂/栓剂
  singleDose: Number,
  doseUnit: String,          // ml/mg/粒
  administrationTime: Date,
  route: String,             // 口服/外用/栓剂
  prescribedBy: String,      // 医生处方/OTC自购
  reason: String,
  note: String,
  createdAt: Date
}
```

**用药间隔提醒**

| 药物 | 最短间隔 | 每日最大次数 | 提醒规则 |
|------|---------|------------|---------|
| 布洛芬 | 6小时 | 4次 | 距上次用药不足6h禁止记录并提醒 |
| 对乙酰氨基酚 | 4小时 | 5次 | 距上次用药不足4h禁止记录并提醒 |
| 抗生素类 | 按医嘱 | 按医嘱 | 提醒按时、按疗程服药 |

#### 3.4 常见药品剂量计算器（基础版）

**V2.0包含药品**
1. 布洛芬（退烧药）
2. 对乙酰氨基酚（退烧药）

**布洛芬剂量计算器**

**适用对象**：6月龄以上儿童

**计算依据**
- 推荐剂量：5-10 mg/kg/次
- 给药间隔：每6-8小时一次
- 每日最大剂量：40 mg/kg/天（不超过1200mg/天）
- 每日最多4次

**常见制剂**
- 美林混悬滴剂：40mg/ml
- 美林混悬液：20mg/ml
- 布洛芬颗粒：200mg/包

**计算器界面**
```
输入项：
├── 孩子体重：[    ] kg
├── 制剂选择：[美林混悬液 20mg/ml ▼]
└── 剂量偏好：[标准剂量(~7mg/kg) ▼]

输出项：
├── 推荐单次剂量：XX mg（XX ml）
├── 推荐剂量范围：XX - XX mg
├── 给药间隔：每6-8小时一次
├── 每日最大剂量：XX mg（XX ml）
├── 每日最多次数：4次
└── ⚠️ 请遵医嘱用药，本计算仅供参考
```

**安全保护机制**
- 体重<5kg或>80kg：提示确认
- <6月龄：提示不适用
- 计算结果超过最大剂量：红色警告，限制为最大剂量
- 可一键保存到用药记录

```
3月龄：脊灰灭活疫苗（第2剂）、百白破疫苗（第1剂）
4月龄：脊灰减毒活疫苗（第3剂）、百白破疫苗（第2剂）
5月龄：百白破疫苗（第3剂）
6月龄：乙肝疫苗（第3剂）、A群流脑多糖疫苗（第1剂）
8月龄：麻腮风疫苗（第1剂）、乙脑减毒活疫苗（第1剂）
9月龄：A群流脑多糖疫苗（第2剂）
18月龄：麻腮风疫苗（第2剂）、百白破疫苗（第4剂）、甲肝减毒活疫苗
```

#### 4.3 非免疫规划疫苗建议

| 疫苗 | 建议年龄 | 价格参考 | 建议说明 |
|------|---------|---------|---------|
| 13价肺炎球菌疫苗 | 2-15月龄 | ~700元/剂 | WHO推荐，预防肺炎球菌感染 |
| 轮状病毒疫苗 | 2-6月龄 | ~200-300元/剂 | 预防轮状病毒腹泻 |

#### 4.4 功能设计

**接种记录**
```javascript
// 疫苗接种记录表 (vaccination_records)
{
  recordId: String,
  childId: String,
  vaccineName: String,
  vaccineType: String,       // 免疫规划/非免疫规划
  doseNumber: Number,        // 第几剂
  totalDoses: Number,
  date: Date,
  batchNumber: String,       // 批号（可选）
  site: String,              // 接种部位
  vaccinator: String,        // 接种单位

    type: String,
    severity: String,
    duration: String
  }],
  note: String,
  createdAt: Date
}
```

**智能提醒**
- 根据出生日期自动生成接种日程表
- 接种日前7天+前1天推送提醒
- 延迟未接种提醒（超过建议时间后提示补种）
- 接种后24h/48h提醒记录反应情况

**接种后反应追踪**
- 接种后系统自动提醒家长记录反应
- 常见反应说明（如"接种后低热是正常免疫反应"）
- 需要就医的异常反应提示
- 反应记录可在下次接种时展示给医生参考

### 5. 多孩子管理（升级版）

> **说明**：V1.0已支持基础的多孩档案管理（添加/编辑/删除孩子、资料二次编辑），V2.0在此基础上升级多孩切换体验、积分管理总览、以及跨孩子的通知和审核区分。

#### 5.1 功能描述

支持一个家长账号管理多个孩子（最多5个），每个孩子拥有完全独立的积分和数据系统，同时为家长提供跨孩子的管理总览视图。

#### 5.2 孩子档案管理（V1.0已实现基础版）

- 添加孩子：姓名、性别、出生日期、头像（参见V1.0 4.3节）
- 编辑孩子信息：支持所有字段的二次编辑（参见V1.0 4.3节）
- 删除孩子（需输入家长密码+二次确认）

#### 5.3 多孩切换交互

**快速切换方式**
```
方式一：顶部导航切换
├── 顶部导航栏显示当前孩子头像+名字
├── 点击头像弹出孩子选择浮层
├── 浮层内展示所有孩子头像+名字+当前积分预览
├── 点击目标孩子 → 切换到该孩子的数据视图
└── 切换时有平滑过渡动画（0.3秒）

方式二：左右滑动切换（V2.0新增）
├── 在首页左右滑动可在不同孩子间切换
├── 底部显示分页指示器（小圆点）
└── 适合日常快速切换场景
```

**视觉区分**
- 每个孩子分配不同的主题色（从预设色板中选择或自动分配）
- 切换孩子时，顶部导航栏颜色随之变化
- 头像旁显示孩子昵称，避免混淆

**切换后的状态保持**
- 切换到某个孩子后，记住该孩子的最后浏览页面
- 下次打开App时，默认显示上次查看的孩子

#### 5.4 多孩积分管理总览（家长端）

**家长控制台仪表盘升级**

**各孩子独立的管理入口**
- 点击孩子卡片进入该孩子的详细管理页
- 每个孩子的任务管理、商城管理、积分调整完全独立
- 家长可为不同孩子设置不同的任务难度和积分值

#### 5.5 多孩兑换审核区分

**审核列表改进**
```
兑换审核列表：
├── 每条申请明确标注孩子姓名和头像
│   例：[🧒小明] 申请兑换"一起玩桌游30分钟"（20积分）
│
├── 支持按孩子筛选：[全部] [小明] [小美]
│
├── 批量审核：可同时选中多条申请批量通过
│
└── 审核通过/拒绝后，消息推送到对应孩子的儿童端
```

#### 5.6 多孩通知区分

**通知消息格式**
- 所有通知消息带有孩子标识：`[小明] 今天完成了"自己刷牙"任务`
- 家长端通知支持按孩子筛选查看
- 紧急通知（如兑换审核）在标题中注明孩子名字

**儿童端隔离**
- 每个孩子只能看到自己的通知
- 切换到某个孩子时，只显示该孩子的消息红点

#### 5.7 数据隔离

- 每个孩子独立的任务列表
- 每个孩子独立的积分系统（积分不可在孩子间转移）
- 每个孩子独立的健康数据
- 每个孩子独立的商城奖励
- 每个孩子独立的情绪日记和成长档案

#### 5.8 多孩家庭的奖励共享机制（可选功能）

**共享奖励商城**
- 家长可设置某些奖励为"家庭共享"（如"周末去公园"对所有孩子可见）
- 也可为每个孩子设置专属奖励
- 共享奖励的库存在所有孩子间共用

**公平性提示**
- 如果某个孩子积分显著高于其他孩子，温和提示家长关注任务难度是否均衡
- 不在儿童端展示其他孩子的积分排名（避免不良竞争）
